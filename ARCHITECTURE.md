## 1. Overview

The system is an SRE Incident Triage Agent built on **Hexagonal Architecture (Ports & Adapters)**, orchestrated by a **LangGraph state machine**, and packaged as a Docker Compose stack.

Three logical concerns, four containers:

1. **`sre-agent`** ‚Äî FastAPI app hosting UI + API + LangGraph agent + adapters
2. **`mock-services`** ‚Äî GitLab-Issues-compatible ticket and notification mock (the *default* adapter targets, swappable for real systems via env var)
3. **`app-db`** ‚Äî PostgreSQL for incident persistence (separate logical DB from Langfuse's Postgres)
4. **`langfuse-*`** ‚Äî self-hosted Langfuse observability stack

### 1.1 Architectural pattern: **Hexagonal Architecture + LangGraph orchestration**

The **domain** (business logic of incident triage) does not know that LLMs, HTTP, databases, FAISS, or LangGraph exist. It depends only on **ports** (Python ABCs). Concrete **adapters** implement those ports and are wired at startup by `infrastructure/container.py` based on environment variables.

**LangGraph remains the orchestrator** of the 6-stage pipeline ‚Äî but each node receives its dependencies as port interfaces, not concrete clients. This means the graph code is identical whether the LLM is Gemini or Anthropic, whether tickets go to a mock or to GitLab, whether storage is in-memory or Postgres.

**Why this pattern:**
- Same Docker Compose runtime as v1 (3 host ports, single deploy unit) ‚Äî **no operational overhead added**
- Swap mock ‚Üí production = environment variable change, **zero code change**
- Domain is unit-testable without booting FastAPI, LangGraph, or any adapter
- Resilience patterns (circuit breaker, fallback, retry) live in *one* place per port ‚Äî not scattered

**Rejected alternatives:**

| Pattern | Why rejected |
|---|---|
| Pure modular monolith (v1) | Business logic tangled with framework calls. Cannot swap LLM/ticketing without editing nodes. |
| Microservices per stage | 48h is not enough wiring time. No scaling need at hackathon scale. |
| Clean Architecture (4 layers) | Same benefits as Hexagonal but heavier vocabulary. Hexagonal is the minimal version of the same idea. |
| Workflow engine (Temporal/Airflow) | Operational overhead. LangGraph already gives a typed DAG with native tracing. |

## 4. Observability Contract v2

> Source of truth for `@developer`. v2 (2026-04-07) restructures observability into **3 layers**: infra spans (per stage), LLM attributes (per call), and Agent behavior attributes (per agent invocation). All three feed Langfuse for traces and Prometheus for metrics.
>
> See [`docs/diagrams/observability-layers.md`](docs/diagrams/observability-layers.md).

### 4.1 Layer 1 ‚Äî Infra spans (per stage)

Span naming convention: `sre.agent.<name>.<phase>` (e.g. `sre.agent.triage.analysis`).

| Stage | Span name | Required attributes |
|---|---|---|
| 1. Ingest | `sre.agent.intake_guard.ingest` | `incident_id`, `has_image: bool`, `has_log: bool`, `text_length: int` |
| 2. Guardrails | `sre.agent.intake_guard.guardrails` | `incident_id`, `injection_detected: bool`, `score: float`, `blocked_reason?: str` |
| 3. Triage | `sre.agent.triage.analysis` | `incident_id`, `context_docs: list[str]`, `severity: str` (+ all Layer 2 attrs + all Layer 3 attrs) |
| 4. Ticket | `sre.agent.integration.ticket_create` | `incident_id`, `ticket_id: str`, `ticket_provider: str`, `severity: str` |
| 5. Notify Team | `sre.agent.integration.notify_team` | `incident_id`, `ticket_id`, `notify_provider: str`, `recipients_count: int` |
| 6. Resolve Notify | `sre.agent.resolution.notify_reporter` | `incident_id`, `ticket_id`, `reporter_email: str` |

All spans share the same root trace identified by `incident_id`. Langfuse `session_id = incident_id`.

### 4.2 Layer 2 ‚Äî LLM attributes (per LLM call)

Every span that wraps a call to `ILLMProvider` MUST emit:

| Attribute | Type | Description |
|---|---|---|
| `llm.cost_usd` | float | Estimated cost = tokens √ó provider price |
| `llm.prompt_name` | str | Logical prompt id from `PROMPT_REGISTRY` |
| `llm.prompt_version` | str | Semver of the prompt template (e.g. `1.0.0`) |
| `llm.tokens_in` | int | Input token count |
| `llm.tokens_out` | int | Output token count |
| `llm.provider_used` | str | Which provider actually responded (`gemini`, `openrouter`, ‚Ä¶) |
| `llm.fallback_used` | bool | True if circuit breaker fell over to fallback |

### 4.3 Layer 3 ‚Äî Agent behavior attributes (per agent invocation)

Every agent subgraph MUST emit a root span with:

| Attribute | Type | Description |
|---|---|---|
| `agent.name` | str | One of `intake_guard`, `triage`, `integration`, `resolution` |
| `agent.iterations` | int | Loop iterations consumed (ReAct steps) |
| `agent.max_iterations` | int | Configured upper bound |
| `agent.rag_queries` | int | Number of `IContextProvider.search_context` calls |
| `agent.rag_hits` | int | Number of context docs returned (sum across queries) |
| `agent.escalated` | bool | True if `should_escalate` returned True |
| `agent.tool_calls` | list[str] | Names of tools invoked (`search_context`, `create_ticket`, ‚Ä¶) |

### 4.4 Quality signals (orchestrator root span)

The orchestrator's root span (one per incident) MUST emit, in addition to all child attributes:

| Attribute | Type | Description |
|---|---|---|
| `triage.severity` | str | `CRITICAL` \| `HIGH` \| `MEDIUM` \| `LOW` |
| `triage.confidence` | str | `high` \| `medium` \| `low` |
| `triage.quality_score` | float | 0.0‚Äì1.0, populated by LLM-judge in eval mode |
| `triage.affected_components` | list[str] | Components identified by triage |
| `case.status_final` | str | Terminal `CaseState.status` |
| `case.total_duration_ms` | int | Wall clock from ingest to terminal status |

### 4.5 Evaluation Layer

> See [`docs/diagrams/eval-pipeline.md`](docs/diagrams/eval-pipeline.md).

The system ships with an offline **LLM-as-judge evaluation pipeline** that runs against a curated golden dataset on every PR (CI hard gate, see ARC-016).

**Components:**
- **Golden dataset** ‚Äî `services/sre-agent/evals/golden_dataset.jsonl` ‚Äî curated cases with expected outcomes
- **Eval runner** ‚Äî invokes the Triage agent through `ILLMProvider` (no shortcuts) for each case
- **TriageJudge** ‚Äî LLM-as-judge that scores each output against the expected
- **Langfuse Dataset Experiment** ‚Äî every run is logged as a Dataset Run in Langfuse for trend analysis
- **CI gate** ‚Äî exit code non-zero if thresholds are breached ‚Üí PR blocked

**Golden case schema:**
```json
{
  "id": "golden-NNN",
  "description": "...",
  "input": {
    "text": "...",
    "modality": "text|log|image",
    "reporter_email": "..."
  },
  "expected": {
    "severity": "CRITICAL|HIGH|MEDIUM|LOW",
    "affected_components": ["..."],
    "root_cause_keywords": ["..."],
    "should_block": false
  }
}
```

**EvalScore schema (returned per case):**
```python
@dataclass(frozen=True)
class EvalScore:
    case_id: str
    severity_correct: bool
    severity_score: float       # 1.0 if correct else 0.0
    root_cause_score: float     # keywords_found / keywords_expected
    components_score: float     # jaccard(expected, predicted)
    overall_score: float        # 0.4*severity + 0.4*root_cause + 0.2*components
    judge_reasoning: str
    blocked_correctly: bool | None   # only for adversarial cases
```

**CI thresholds (hard gate, ARC-016):**

| Metric | Threshold | Failure mode |
|---|---|---|
| `avg_overall_score` | `>= 0.70` | CI fails ‚Üí PR blocked |
| `adversarial_recall` | `== 1.0` (100%) | CI fails ‚Üí PR blocked |

The eval runner is owned by `@developer`. The thresholds and golden dataset shape are owned by `@architect` ‚Äî modifying either requires architect approval.

### 4.6 Prometheus Metrics Contract

The agent exposes `GET /metrics` in Prometheus text format. Naming convention: `sre_<noun>_<unit>` with OpenMetrics suffix (`_total`, `_seconds`, `_usd`).

**Counters:**
```
sre_incidents_received_total
sre_incidents_by_severity_total{severity}
sre_incidents_escalated_total{agent_name}
sre_llm_cost_usd_total{provider}
sre_llm_fallback_activations_total{from_provider, to_provider}
sre_llm_calls_total{provider, prompt_name}
```

**Histograms:**
```
sre_agent_iterations_bucket{agent_name}
sre_triage_quality_score_bucket
sre_pipeline_duration_seconds_bucket
```

**Gauges:**
```
sre_rag_hit_rate{context_provider}
sre_active_incidents
```

**Endpoint policy:**
- Path: `GET /metrics`
- Format: Prometheus text exposition format (handled by `prometheus-client`)
- Auth in dev: **none** (judges and operators must be able to scrape easily)
- Auth in prod: **basic auth** toggled by `METRICS_AUTH_ENABLED=true` env var; credentials via `METRICS_BASIC_USER` / `METRICS_BASIC_PASSWORD`

### 4.7 Structured logs (JSON)

Every log line MUST include `ts`, `level`, `incident_id`, `stage`, `event`, `duration_ms`. Mandatory event names: `incident.received`, `guardrails.evaluated`, `guardrails.blocked`, `triage.started`, `triage.completed`, `triage.fallback_used`, `triage.degraded_mode`, `ticket.created`, `notify.team.sent`, `notify.email.sent`, `resolve.notified`, `pipeline.failed`.

### 4.8 Correlation ID

- Generated at ingest as UUID v7 ‚Üí assigned as `incident_id`
- Propagated as HTTP header `X-Incident-ID` between containers
- Returned to the UI so it can link to the Langfuse trace

### 4.9 Health checks

- `GET /health` ‚Üí `{"status":"ok","stage_count":6,"adapters":{...},"langfuse":"connected|disabled"}`
- The `adapters` block lists which concrete adapter is bound to each port ‚Äî judges and ops both want to see this.

| Concept | Pattern | Example |
|---|---|---|
| Span name | `sre.agent.<name>.<phase>` | `sre.agent.triage.analysis` |
| Prometheus metric | `sre_<noun>_<unit>` + OpenMetrics suffix | `sre_llm_cost_usd_total` |

## 5. Architecture Rules ‚Äî DO NOT BREAK

These are enforced by `@architect` review. Any violation fails the gate.

1. **Domain purity.** Files under `app/domain/` MUST NOT import anything outside the standard library, `pydantic`, or other `app/domain/` files. No FastAPI, no LangGraph, no httpx, no SDKs.
2. **Agents depend on ports, not adapters.** Files under `app/orchestration/agents/` and `app/orchestration/orchestrator/` MUST import from `app/domain/ports/` only ‚Äî never from `app/adapters/`. The `container` injects concrete adapters.
3. **The orchestrator graph in `app/orchestration/orchestrator/graph.py` is the ONLY top-level coordinator.** API routes invoke `build_orchestrator_graph()` (sync flow) or `build_resolution_graph()` (async webhook flow); they never coordinate agents directly.
4. **Adapters are leaves.** Files under `app/adapters/` MUST NOT import from `app/orchestration/`, `app/api/`, or other adapter packages. They depend on `app/domain/ports/` (to implement) and external libraries.
5. **Every stage MUST emit a Langfuse span and a structured log line** with `incident_id`. (See section 4.)
6. **No LLM call may use raw user input as instructions.** All user input passes through `app/security/` first AND is wrapped in clearly delimited content blocks.
7. **Tool/port calls accept only typed Pydantic args.** No free-form URLs, shell strings, or filenames flow from LLM into adapter execution.
8. **Secrets only via environment variables.** Never commit `.env`. Never hardcode keys.
9. **No new dependencies without `@architect` approval.** All deps live in `services/sre-agent/requirements.txt`.
10. **Only the host ports declared in `docker-compose.yml` may be exposed.** Adding ports requires `@architect` approval. Today: `8000` (sre-agent), `9000` (mock-services), `3000` (Langfuse). `app-db` is internal-only.
11. **Adapter selection is config, never code.** A new provider is added by writing a new adapter class and registering it in `infrastructure/container.py` ‚Äî never by editing nodes, services, or the graph.
12. **Output schemas are Pydantic models.** LLMs return structured output via the `ILLMProvider.triage()` contract. Never parsed regex.
13. **(ARC-012) Single writer for CaseState.** Only the orchestrator may mutate `CaseState`. Agents receive an immutable Projection (`IntakeProjection`, `TriageProjection`, `IntegrationProjection`, `ResolutionProjection`) and MUST return exactly one `AgentEvent`. The orchestrator folds the event into CaseState. This guarantees a single source of truth and a replayable audit log.
14. **(ARC-013) Agents are subgraphs, not free-form code.** Each agent under `app/orchestration/agents/<name>/` exposes a `build_<name>_agent(container)` factory that returns a compiled LangGraph. The orchestrator composes them via `add_node` ‚Äî never by importing internal agent functions.
15. **(ARC-014) Resolution is async-only.** The Resolution Agent MUST NOT be wired into the synchronous incident graph. It is exposed via `build_resolution_graph()` and triggered by the resolution webhook handler. Rationale: the sync API response must never block on a human/external resolution event (DEC-006).
16. **(ARC-015) Prompt registry is mandatory.** Every LLM prompt MUST be resolved through `app/llm/prompt_registry.py::PROMPT_REGISTRY`. Inline prompt strings inside `app/orchestration/agents/**` are **forbidden**. Each registry entry carries a versioned id (`<purpose>-v<major>.<minor>.<patch>`) which is emitted as `llm.prompt_name` / `llm.prompt_version` on every LLM span (see ¬ß4.2). Reviewer rejects on sight any agent file containing a multiline string passed to `ILLMProvider`.
17. **(ARC-016) Evals are a CI hard gate.** The eval runner (see ¬ß4.5) MUST execute on every pull request via GitHub Actions. A PR MUST be blocked from merge if `avg_overall_score < 0.70` OR if any adversarial case is not blocked (`adversarial_recall < 1.0`). The CI job `eval` is non-skippable. Threshold changes require `@architect` approval and a corresponding update to this section.

### 6.2.2 State Contracts

Two distinct state types ‚Äî they MUST NOT be confused:

| Type | Owner | Lifetime | Mutability |
|---|---|---|---|
| **`CaseState`** (`orchestrator/state.py`) | Orchestrator only | One incident, full lifecycle | Mutable, single writer |
| **`AgentLocalState`** (`shared/agent_state.py`) | One agent subgraph | One subgraph invocation | Mutable inside the subgraph, dies on return |
| **`<X>Projection`** (frozen dataclass in `orchestrator/state.py`) | Built by orchestrator, read by agent | One subgraph invocation | **Immutable** |
| **`AgentEvent`** (frozen dataclass) | Returned by agent, applied by orchestrator | Persisted in `CaseState.events` | **Immutable** |

**Flow:**
1. Orchestrator builds `XProjection` from `CaseState`.
2. Orchestrator invokes agent subgraph with `{"projection": proj}`.
3. Agent runs internally on its own `AgentLocalState`.
4. Agent returns `final_event: AgentEvent`.
5. Orchestrator folds `AgentEvent` into `CaseState` (the **only** mutation point).

### 6.2.3 CaseState lifecycle

See `docs/diagrams/state-case-lifecycle.md`. Status enum:

```
NEW ‚Üí INTAKE_OK ‚Üí TRIAGED ‚Üí TICKETED ‚Üí NOTIFIED ‚Üí AWAITING_RESOLUTION ‚Üí RESOLVED
  ‚îÇ
  ‚îî‚îÄ‚ñ∫ INTAKE_BLOCKED   (terminal)
  ‚îî‚îÄ‚ñ∫ FAILED            (terminal, on agent.error)
```

### 6.2.4 Escalation policy

Lives in `orchestrator/router.py` ‚Äî pure functions only, no I/O. To change routing logic, edit this file and only this file.

| Routing decision | Function | Current policy |
|---|---|---|
| After intake | `route_after_intake` | `INTAKE_BLOCKED` ‚Üí END, otherwise ‚Üí triage |
| After triage | `route_after_triage` | `error` ‚Üí END, otherwise ‚Üí integration |
| After integration | `route_after_integration` | always ‚Üí END (resolution is async) |
| `should_escalate(state)` | reserved | hackathon: always False |

### 6.2.5 Why two compiled graphs (DEC-006 / ARC-014)

The synchronous incident graph (`build_orchestrator_graph`) returns within bounded time ‚Äî its SLO is API response latency. Resolution depends on external human/system action of unknown duration, so it lives in a separate graph (`build_resolution_graph`) invoked by the resolution webhook handler. The two graphs share `CaseState` schema but never share a single invocation.

## 10. Security Architecture ‚Äî Prompt Injection Defense (5 layers)

| Layer | What it does | Where |
|---|---|---|
| 1. Input sanitization | Strip control chars, zero-width Unicode. Length limit 8000. File allowlist. Size limit 5MB. | `app/security/input_sanitizer.py` |
| 2. Heuristic detection | Regex against known attack patterns | `app/security/prompt_injection.py` |
| 3. LLM judge | Cheap classification via `ILLMProvider.classify_injection()` | `app/security/prompt_injection.py` |
| 4. Output schema enforcement | Pydantic structured output via `ILLMProvider.triage()`. Free-form text never drives tool calls. | `domain/ports/llm_provider.py` + adapters |
| 5. Tool sandboxing | Adapters accept typed Pydantic args. No shell, no dynamic URLs. | `adapters/**/*.py` |

Blocked incidents are persisted with `blocked=true`, traced with `injection_detected=true`, and shown in the UI as "Blocked by guardrails ‚Äî see trace".

## 11. Persistence Schema (PostgreSQL)

Logical database `sre_agent` on the `app-db` Postgres instance. Managed by SQLAlchemy + Alembic.

```sql
CREATE TABLE incidents (
  id              UUID PRIMARY KEY,
  reporter_email  TEXT NOT NULL,
  title           TEXT NOT NULL,
  description     TEXT NOT NULL,
  status          TEXT NOT NULL,        -- received|triaging|ticketed|resolved|blocked|failed
  severity        TEXT,                 -- P1|P2|P3|P4
  blocked         BOOLEAN DEFAULT FALSE,
  blocked_reason  TEXT,
  has_image       BOOLEAN DEFAULT FALSE,
  has_log         BOOLEAN DEFAULT FALSE,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE tickets (
  id              TEXT PRIMARY KEY,     -- provider-specific id
  incident_id     UUID NOT NULL REFERENCES incidents(id),
  provider        TEXT NOT NULL,        -- mock|gitlab|jira
  url             TEXT,
  status          TEXT NOT NULL,        -- open|resolved
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  resolved_at     TIMESTAMPTZ
);

CREATE TABLE incident_files (
  id              UUID PRIMARY KEY,
  incident_id     UUID NOT NULL REFERENCES incidents(id),
  filename        TEXT NOT NULL,
  content_type    TEXT NOT NULL,
  size_bytes      INTEGER NOT NULL,
  storage_path    TEXT NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_incidents_status ON incidents(status);
CREATE INDEX idx_tickets_incident ON tickets(incident_id);
```

`MemoryStorageAdapter` mirrors the same shape using Python dicts ‚Äî used for unit tests and the `STORAGE_PROVIDER=memory` ultra-fast demo mode.
