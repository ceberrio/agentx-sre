## 1. Overview

The system is an **SRE Incident Triage Platform** built on **Hexagonal Architecture (Ports & Adapters)**, orchestrated by a **LangGraph state machine**, and packaged as a Docker Compose stack.

### 1.1 Platform structure (v4 ŌĆö two-tier)

The platform is composed of two logical tiers:

**Platform layer (new in v4):**
- `sre-web` ŌĆö React 18 SPA (TypeScript + Vite + Tailwind + SoftServe Design System) served via Nginx. Provides the full platform UI: authentication, incident management, configuration, dashboard.

**Agent layer (original):**
- `sre-agent` ŌĆö FastAPI backend. Hosts the LangGraph agent pipeline, all domain ports, adapters, the REST API (consumed by `sre-web`), and the Prometheus `/metrics` endpoint.
- `mock-services` ŌĆö GitLab-Issues-compatible ticket and notification mock (default adapter, swappable via DB config).
- `app-db` ŌĆö PostgreSQL for incident persistence, RBAC users, LLM config, platform config, governance thresholds (all in separate tables, same DB).
- `langfuse-*` ŌĆö self-hosted Langfuse observability stack.

### 1.2 Platform overview

**Five containers in total (v4):**

1. **`sre-web`** ŌĆö React SPA (Nginx). Exposed on host port `5173`. No Python, no business logic. Calls `sre-agent` API at `http://localhost:8000`. *(NEW ŌĆö v4)*
2. **`sre-agent`** ŌĆö FastAPI + LangGraph + all adapters. Exposed on host port `8000`. Now also serves: auth endpoints, RBAC middleware, platform config API, LLM hot reload, eShop indexing.
3. **`mock-services`** ŌĆö GitLab-Issues-compatible mock. Exposed on host port `9000`.
4. **`app-db`** ŌĆö PostgreSQL (internal only). Stores: incidents, tickets, files, governance thresholds, users, llm_config, platform_config.
5. **`langfuse-*`** ŌĆö Self-hosted Langfuse. `langfuse-web` exposed on host port `3000`.

**Host-exposed ports (v4):** `5173` (sre-web), `8000` (sre-agent), `9000` (mock-services), `3000` (Langfuse). See ARC-020 and DEC-A10 for port governance.

### 1.3 Architectural pattern: **Hexagonal Architecture + LangGraph orchestration**

The **domain** (business logic of incident triage) does not know that LLMs, HTTP, databases, FAISS, or LangGraph exist. It depends only on **ports** (Python ABCs). Concrete **adapters** implement those ports and are wired at startup by `infrastructure/container.py` based on environment variables and ŌĆö from v4 ŌĆö DB-stored configuration.

**LangGraph remains the orchestrator** of the 6-stage pipeline. The React SPA is a pure consumer of the FastAPI REST API ŌĆö it never calls adapters directly.

**Why this pattern:**
- Same Docker Compose runtime as v1 (minimal host ports, single deploy unit) ŌĆö **no operational overhead added**
- Swap mock ŌåÆ production = DB config change, **zero code change**
- Domain is unit-testable without booting FastAPI, LangGraph, or any adapter
- Resilience patterns (circuit breaker, fallback, retry) live in *one* place per port ŌĆö not scattered
- React SPA is decoupled from the Python backend ŌĆö can be developed and built independently

**Rejected alternatives:**

| Pattern | Why rejected |
|---|---|
| Pure modular monolith (v1) | Business logic tangled with framework calls. Cannot swap LLM/ticketing without editing nodes. |
| Microservices per stage | 48h is not enough wiring time. No scaling need at hackathon scale. |
| FastAPI serving React build (DEC-A01 option b) | Couples Node build tooling into Python container. Complicates Dockerfile and CI. Nginx is the right tool for serving SPAs. |
| Workflow engine (Temporal/Airflow) | Operational overhead. LangGraph already gives a typed DAG with native tracing. |

### 4.1 Layer 1 ŌĆö Infra spans (per stage)

|---|---|---|
| 1. Ingest | `sre.agent.intake_guard.ingest` | `incident_id`, `has_image: bool`, `has_log: bool`, `text_length: int` |
| 2. Guardrails | `sre.agent.intake_guard.guardrails` | `incident_id`, `injection_detected: bool`, `score: float`, `blocked_reason?: str` |

### 4.5 Evaluation Layer

> See [`docs/diagrams/eval-pipeline.md`](docs/diagrams/eval-pipeline.md).

The system ships with an offline **LLM-as-judge evaluation pipeline** that runs against a curated golden dataset on every PR (CI hard gate, see ARC-016).

**Components:**
- **Golden dataset** ŌĆö `services/sre-agent/evals/golden_dataset.jsonl` ŌĆö curated cases with expected outcomes
- **Eval runner** ŌĆö invokes the Triage agent through `ILLMProvider` (no shortcuts) for each case
- **TriageJudge** ŌĆö LLM-as-judge that scores each output against the expected
- **Langfuse Dataset Experiment** ŌĆö every run is logged as a Dataset Run in Langfuse for trend analysis
- **CI gate** ŌĆö exit code non-zero if thresholds are breached ŌåÆ PR blocked

**Golden case schema:**
```json
{
  "id": "golden-NNN",
  "description": "...",
  "input": {
    "text": "...",
    "modality": "text|log|image",
    "reporter_email": "..."
  },
  "expected": {
    "severity": "CRITICAL|HIGH|MEDIUM|LOW",
    "affected_components": ["..."],
    "root_cause_keywords": ["..."],
    "should_block": false
  }
}
```

**EvalScore schema (returned per case):**
```python
@dataclass(frozen=True)
class EvalScore:
    case_id: str
    severity_correct: bool
    severity_score: float       # 1.0 if correct else 0.0
    root_cause_score: float     # keywords_found / keywords_expected
    components_score: float     # jaccard(expected, predicted)
    overall_score: float        # 0.4*severity + 0.4*root_cause + 0.2*components
    judge_reasoning: str
    blocked_correctly: bool | None   # only for adversarial cases
```

**CI thresholds (hard gate, ARC-016):**

| Metric | Threshold | Failure mode |
|---|---|---|
| `avg_overall_score` | `>= 0.70` | CI fails ŌåÆ PR blocked |
| `adversarial_recall` | `== 1.0` (100%) | CI fails ŌåÆ PR blocked |

The eval runner is owned by `@developer`. The thresholds and golden dataset shape are owned by `@architect` ŌĆö modifying either requires architect approval.

### 4.11 Governance Thresholds Contract

> Binding contract between `IGovernanceProvider` (producer) and `router.py` / Integration agent (consumers).
> Source of truth: table `governance_thresholds` in `app-db` PostgreSQL. The UI at `governance.html` is the operator interface.

**Table schema ŌĆö `governance_thresholds`:**
```sql
CREATE TABLE governance_thresholds (
  key         TEXT PRIMARY KEY,
  value_text  TEXT NOT NULL,         -- canonical serialization; adapter casts to Python type
  value_type  TEXT NOT NULL,         -- 'float' | 'bool' | 'int' | 'str'
  description TEXT,
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_by  TEXT                   -- operator email or 'system'
);
```

**`GovernanceThresholds` model (owned by `app/domain/entities/governance.py`):**
```python
class GovernanceThresholds(BaseModel):
    confidence_escalation_min: float = 0.60
    # Below this ŌåÆ human review required (should_escalate returns True)

    quality_score_min_for_autoticket: float = 0.65
    # Below this ŌåÆ ticket flagged with needs_review=True

    severity_autoticket_threshold: str = "LOW"
    # Severities at or above this always auto-ticket (e.g. "LOW" = all; "MEDIUM" = skip LOW)

    kill_switch_enabled: bool = False
    # When True ŌåÆ ALL incidents are escalated; automated ticket generation paused

    max_rag_docs_to_expose: int = 5
    # Maximum RAG documents included in ExplainabilityReport.rag_attributions
```

**Cache policy:**
- The `PostgresGovernanceAdapter` caches the full `GovernanceThresholds` object in-memory.
- TTL controlled by `GOVERNANCE_CACHE_TTL_S` env var (default: `60` seconds).
- A `PUT /governance/thresholds` call invalidates the cache immediately.
- The `MemoryGovernanceAdapter` holds thresholds in a dict; TTL is irrelevant (always fresh).

**`IGovernanceProvider` port contract (owned by `app/domain/ports/governance_provider.py`):**
```python
class IGovernanceProvider(ABC):
    name: str  # "postgres" | "memory"

    async def get_thresholds(self) -> GovernanceThresholds: ...
    # Returns cached or freshly loaded thresholds. Never raises ŌĆö returns defaults on DB error.

    async def update_threshold(self, key: str, value: str) -> GovernanceThresholds: ...
    # Updates one field in DB, invalidates cache, returns the updated GovernanceThresholds.
    # Raises ValueError if key is not a valid GovernanceThresholds field name.
    # Raises TypeError if value cannot be cast to the field's type.
```

**`should_escalate()` redesign ŌĆö `app/orchestration/orchestrator/router.py`:**

The current stub (`return False`) MUST be replaced. New signature and policy:

```python
def should_escalate(state: CaseState, thresholds: GovernanceThresholds) -> tuple[bool, str | None]:
    """Returns (should_escalate: bool, reason: str | None).
    reason is one of: 'low_confidence' | 'kill_switch' | 'severity_threshold' | None.
    Pure function ŌĆö no I/O. GovernanceThresholds injected by the orchestrator.
    """
    triage = state.get("triage_result")
    if thresholds.kill_switch_enabled:
        return True, "kill_switch"
    if triage and triage.confidence < thresholds.confidence_escalation_min:
        return True, "low_confidence"
    if triage and _severity_at_or_above(triage.severity, thresholds.severity_autoticket_threshold):
        return False, None  # high severity = auto-ticket, NOT escalation
    return False, None
```

> `@developer` implements this function. `@architect` owns the policy table above.
> The orchestrator must call `IGovernanceProvider.get_thresholds()` once per incident (at graph entry) and pass the result as a parameter to routing functions.

**API endpoints (owned by `app/api/routes_governance.py`):**

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/governance/thresholds` | API key | Returns current `GovernanceThresholds` as JSON |
| `PUT` | `/governance/thresholds` | API key | Updates one or more fields; body: `{"key": "...", "value": "..."}` (single) or `{"updates": [{...}]}` (batch); invalidates cache |

**UI:** `app/ui/templates/governance.html` ŌĆö HTMX page. Operators view all thresholds and submit inline edits. Each field shows its current value, type, description, and last-updated timestamp. Kill switch has a confirmation modal.

### 4.14 Config-from-DB Contract (v4)

> **Principle (established v4 ŌĆö 2026-04-08):** After initial bootstrap (DB connection, app environment, Langfuse host), all system behavior is governed by database-stored configuration managed through the platform UI (`/config`). `.env` contains only infrastructure bootstrap variables. No business logic value may be hardcoded in Python files or `.env` after HU-P032 is implemented.

#### 4.14.2 DB-managed configuration ŌĆö table structure (DEC-A04)

**Decision DEC-A04:** Separate port structure ŌĆö `IGovernanceProvider` keeps exclusive ownership of governance thresholds (`governance_thresholds` table). Two new ports are created:
- **`ILLMConfigProvider`** ŌĆö owns the `llm_config` table (LLM provider, model, API keys, circuit breaker params)
- **`IPlatformConfigProvider`** ŌĆö owns the `platform_config` table (all other business variables: ticket, notify, context, security, observability, storage)

Rationale: single-responsibility per port, no breaking change to existing `IGovernanceProvider` (HU-012 implementation plan unchanged), clean separation between governance business logic and infrastructure adapter configuration.

**Three DB tables (see ┬¦11 for full SQL schemas):**

| Table | Port | Variables managed | Hot reload? |
|---|---|---|---|
| `governance_thresholds` | `IGovernanceProvider` | 5 governance thresholds (confidence, quality score, severity threshold, kill switch, max RAG docs) | Yes ŌĆö TTL cache invalidated on PUT |
| `llm_config` | `ILLMConfigProvider` | Provider, model, API keys (encrypted), fallback provider/model, circuit breaker params | Yes ŌĆö < 5 seconds (DEC-A06) |
| `platform_config` | `IPlatformConfigProvider` | Ticket provider + credentials, notify provider + credentials, context source, eShop config, security params, observability params, storage backend, allowed file types | Partial ŌĆö most hot-reload; eShop URL requires restart |

#### 4.14.3 Cold-start seed strategy (DEC-A05)

**Strategy:** Alembic migration seed ŌĆö idempotent `INSERT ... ON CONFLICT DO NOTHING` for all rows in all three config tables. The seed runs as part of `alembic upgrade head` (already executed in the `sre-agent` container startup command).

**Migration structure:**
- Existing migration: seeds `governance_thresholds` (5 rows) ŌĆö no change
- New migration A: creates `llm_config` table + seeds default primary/fallback rows
- New migration B: creates `platform_config` table + seeds all keys with documented defaults
- New migration C: creates `users` table + seeds 5 demo users (one per role) for hackathon

**Behavior when DB is unavailable at startup:**
1. The FastAPI app fails to start ŌĆö `alembic upgrade head` exits non-zero.
2. Docker Compose `depends_on: app-db: condition: service_healthy` prevents startup before Postgres is ready.
3. If the DB becomes unavailable AFTER startup: `IGovernanceProvider`, `ILLMConfigProvider`, and `IPlatformConfigProvider` each return their in-memory cached values. If no cache exists, they return Pydantic model defaults. They MUST NOT raise ŌĆö see ARC-020.

**Rule:** The system must never fail to triage an incident because the config DB is temporarily unavailable. All providers must have safe in-memory defaults.

#### 4.14.4 LLM hot reload mechanism (DEC-A06)

**Decision:** Option (c) ŌĆö **cache with explicit TTL invalidation**. The `container.py` holds a singleton reference to the active `LLMCircuitBreaker`. An `asyncio.Lock`-protected `reload_llm_adapter()` coroutine replaces the adapter atomically.

**Mechanism:**
1. `PUT /config/llm` saves to `llm_config` table.
2. The route handler calls `container.reload_llm_adapter()` after a successful DB write.
3. `reload_llm_adapter()` acquires an `asyncio.Lock`, instantiates the new primary and fallback adapters from the just-saved config, wraps them in a new `LLMCircuitBreaker`, and atomically replaces the singleton reference.
4. Concurrent requests that started before the reload complete with the old adapter (safe ŌĆö atomically replaced reference). New requests after the lock releases use the new adapter.
5. Total propagation time: < 1s (no polling needed). SLO: < 5s end-to-end (AC-06 / SA-012).

**Circuit breaker reconfiguration (DEC ŌĆö circuit breaker hot reload):**
The `LLMCircuitBreaker` must expose `reconfigure(threshold: int, cooldown: int)` method that updates the parameters in-place **without losing the current failure count**. If the circuit breaker is currently `OPEN`, it continues open until the previous cooldown expires, then uses the new cooldown for subsequent states.

**Rule (ARC-020-derived):** The `reload_llm_adapter()` call is the ONLY mechanism for changing the active LLM adapter. No other code path may replace the adapter. The route handler is responsible for calling it after a successful DB write.

#### 4.15.3 JWT structure

JWT signed with HS256 using `JWT_SECRET` env var. Token lifetime: 24 hours.

```json
{
  "sub": "user@example.com",
  "role": "operator",
  "name": "Full Name",
  "iat": 1712600000,
  "exp": 1712686400
}
```

**Claims:**
- `sub` ŌĆö user email (must match a row in the `users` table where `is_active = true`)
- `role` ŌĆö one of `superadmin`, `admin`, `flow_configurator`, `operator`, `viewer`
- `name` ŌĆö display name from the `users` table
- `iat` / `exp` ŌĆö standard JWT timestamps

#### 4.15.4 Mock Google OAuth flow (hackathon)

The system simulates Google OAuth without a real Google Cloud project.

**Endpoint:** `GET /auth/google/callback?email={email}`

**Flow:**
1. React login page shows a dropdown with 5 pre-seeded demo users (one per role).
2. User selects a demo user and clicks "Sign in with Google".
3. React calls `GET /auth/google/callback?email={selected_email}`.
4. FastAPI looks up the email in the `users` table.
5. If found and `is_active=true`: returns `{"access_token": "...", "token_type": "Bearer"}`.
6. If not found or inactive: returns HTTP 401 `{"detail": "user_not_authorized"}`.
7. React stores the token in **`localStorage`** (DEC-A02 decision ŌĆö see ┬¦15).
8. All subsequent API calls include `Authorization: Bearer {token}` header.

**JWT storage ŌĆö DEC-A02:** `localStorage`. Rationale: for the hackathon mock auth context, the simplicity and CSRF immunity of `localStorage` outweigh the XSS risk. The mock auth does not protect real credentials. A production deployment would migrate to `httpOnly` cookies with `SameSite=Strict`. This is documented as a known security trade-off (SA-003).

#### 4.15.5 RBAC enforcement implementation

```python

from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
import jwt  # python-jose

security = HTTPBearer()

async def get_current_user(
    credentials: HTTPAuthorizationCredentials = Depends(security)
) -> dict:
    """Decode and validate JWT. Returns payload dict."""
    ...

def require_role(*roles: str):
    """Returns a FastAPI dependency that enforces role membership."""
    async def _dep(user: dict = Depends(get_current_user)) -> dict:
        if user["role"] not in roles:
            raise HTTPException(status_code=403, detail="insufficient_role")
        return user
    return _dep

# Usage in route:

# async def update_llm_config(
#     _: dict = Depends(require_role("admin", "superadmin")),
#     ...
# ):
```

**Rule (ARC-022):** RBAC is enforced at the FastAPI route level via `require_role()`. The domain layer has no knowledge of roles. The orchestration layer has no knowledge of roles. Putting role checks in agents, services, or adapters is a violation.

## 5. Architecture Rules ŌĆö DO NOT BREAK

These are enforced by `@architect` review. Any violation fails the gate.

1. **Domain purity.** Files under `app/domain/` MUST NOT import anything outside the standard library, `pydantic`, or other `app/domain/` files. No FastAPI, no LangGraph, no httpx, no SDKs.
2. **Agents depend on ports, not adapters.** Files under `app/orchestration/agents/` and `app/orchestration/orchestrator/` MUST import from `app/domain/ports/` only ŌĆö never from `app/adapters/`. The `container` injects concrete adapters.
3. **The orchestrator graph in `app/orchestration/orchestrator/graph.py` is the ONLY top-level coordinator.** API routes invoke `build_orchestrator_graph()` (sync flow) or `build_resolution_graph()` (async webhook flow); they never coordinate agents directly.
4. **Adapters are leaves.** Files under `app/adapters/` MUST NOT import from `app/orchestration/`, `app/api/`, or other adapter packages. They depend on `app/domain/ports/` (to implement) and external libraries.
5. **Every stage MUST emit a Langfuse span and a structured log line** with `incident_id`. (See section 4.)
6. **No LLM call may use raw user input as instructions.** All user input passes through `app/security/` first AND is wrapped in clearly delimited content blocks.
7. **Tool/port calls accept only typed Pydantic args.** No free-form URLs, shell strings, or filenames flow from LLM into adapter execution.
8. **Secrets only via environment variables.** Never commit `.env`. Never hardcode keys.
9. **No new dependencies without `@architect` approval.** All deps live in `services/sre-agent/requirements.txt`.
10. **Only the host ports declared in `docker-compose.yml` may be exposed.** Adding ports requires `@architect` approval. Today: `8000` (sre-agent), `9000` (mock-services), `3000` (Langfuse). `app-db` is internal-only.
11. **Adapter selection is config, never code.** A new provider is added by writing a new adapter class and registering it in `infrastructure/container.py` ŌĆö never by editing nodes, services, or the graph.
12. **Output schemas are Pydantic models.** LLMs return structured output via the `ILLMProvider.triage()` contract. Never parsed regex.
13. **(ARC-012) Single writer for CaseState.** Only the orchestrator may mutate `CaseState`. Agents receive an immutable Projection (`IntakeProjection`, `TriageProjection`, `IntegrationProjection`, `ResolutionProjection`) and MUST return exactly one `AgentEvent`. The orchestrator folds the event into CaseState. This guarantees a single source of truth and a replayable audit log.
14. **(ARC-013) Agents are subgraphs, not free-form code.** Each agent under `app/orchestration/agents/<name>/` exposes a `build_<name>_agent(container)` factory that returns a compiled LangGraph. The orchestrator composes them via `add_node` ŌĆö never by importing internal agent functions.
15. **(ARC-014) Resolution is async-only.** The Resolution Agent MUST NOT be wired into the synchronous incident graph. It is exposed via `build_resolution_graph()` and triggered by the resolution webhook handler. Rationale: the sync API response must never block on a human/external resolution event (DEC-006).
16. **(ARC-015) Prompt registry is mandatory.** Every LLM prompt MUST be resolved through `app/llm/prompt_registry.py::PROMPT_REGISTRY`. Inline prompt strings inside `app/orchestration/agents/**` are **forbidden**. Each registry entry carries a versioned id (`<purpose>-v<major>.<minor>.<patch>`) which is emitted as `llm.prompt_name` / `llm.prompt_version` on every LLM span (see ┬¦4.2). Reviewer rejects on sight any agent file containing a multiline string passed to `ILLMProvider`.
17. **(ARC-016) Evals are a CI hard gate.** The eval runner (see ┬¦4.5) MUST execute on every pull request via GitHub Actions. A PR MUST be blocked from merge if `avg_overall_score < 0.70` OR if any adversarial case is not blocked (`adversarial_recall < 1.0`). The CI job `eval` is non-skippable. Threshold changes require `@architect` approval and a corresponding update to this section.

18. **(ARC-017) Governance thresholds are runtime config, not code.** Business governance thresholds (`confidence_escalation_min`, `quality_score_min_for_autoticket`, `severity_autoticket_threshold`, `kill_switch_enabled`, `max_rag_docs_to_expose`) MUST be stored in the `governance_thresholds` PostgreSQL table and accessed exclusively via `IGovernanceProvider`. Hardcoding any of these values as Python literals in nodes, agents, routers, or services is **forbidden**. The only acceptable hardcoded threshold is the CI eval gate (`avg_overall_score >= 0.70`), which is an architectural decision owned by `@architect` and documented in ┬¦4.5. Reviewer rejects on sight any routing or policy function that reads a governance threshold directly from `Settings` or as a Python literal.

19. **(ARC-018) Every triage result MUST carry explainability data.** `TriageResult` objects returned from `ILLMProvider.triage()` MUST include `rag_attribution` populated from the RAG results passed to the Triage agent. The `IExplainabilityProvider.compute_report()` call MUST complete before the orchestrator emits its root span. An empty `rag_attribution` list is only acceptable when `TriageResult.degraded == True`. Reviewer rejects on sight any `TriageResult` construction in production code with `rag_attribution=[]` unless `degraded=True` is also set.

20. **(ARC-019) Online eval is mandatory in production.** When `app_env == "production"`, the `IExplainabilityProvider.post_online_eval()` MUST be invoked as a background task after the orchestrator root span closes (see ┬¦4.13). Failure to post MUST be logged as an error and counted via `sre_online_eval_post_failures_total` Prometheus counter ŌĆö it MUST NOT raise an exception that breaks the incident response pipeline. In `development` mode this is best-effort: log on failure, do not raise. The `memory_adapter` always silently drops `post_online_eval()` calls (test isolation). Reviewer rejects on sight any removal of the background task invocation in the orchestrator's terminal node.

21. **(ARC-020) No business configuration value may be hardcoded after HU-P032.** After the Config-from-DB platform is deployed, no business configuration value (LLM provider, API keys, ticket system, notify provider, file type allowlists, circuit breaker thresholds, observability settings, security settings) may be hardcoded as a Python literal in source code or as a value in `.env` files. All such values MUST live in one of the three config tables (`governance_thresholds`, `llm_config`, `platform_config`) and be accessed exclusively via the corresponding port (`IGovernanceProvider`, `ILLMConfigProvider`, `IPlatformConfigProvider`). The ONLY permitted `.env` values are the seven bootstrap variables listed in ┬¦4.14.1. Reviewer rejects on sight any new `settings.some_business_variable` reference introduced after HU-P032 is merged. Config providers MUST return safe in-memory defaults when the DB is unreachable ŌĆö they MUST NOT raise.

22. **(ARC-021) React UI text is English-only. SoftServe Design Tokens are the single styling source of truth.** All user-visible strings in the React SPA (labels, placeholders, validation messages, button text, toast messages, section headers, error messages) MUST be in English. No i18n library (react-i18next, lingui, FormatJS) may be installed or used. No locale files (`.json` translations) may be created. All visual styling MUST use the SoftServe Design System tokens documented in ┬¦10 ŌĆö no hardcoded color hex values, no arbitrary Tailwind classes that override the design token palette. Reviewer rejects on sight any Spanish text in React component output or any `text-[#hex]` Tailwind class that bypasses the token system.

23. **(ARC-022) RBAC is enforced exclusively at the FastAPI route layer.** The `require_role(*roles)` dependency defined in `app/api/deps.py` is the ONLY mechanism for enforcing role-based access control. Domain entities, domain services, domain ports, orchestration agents, and adapters MUST NOT contain role-checking logic. UI-level role hiding (hiding buttons, sections, or routes in React) is PERMITTED as a UX enhancement but is NOT a security control ŌĆö the backend enforcement is authoritative. Reviewer rejects on sight any `if user.role == "admin"` conditional inside `app/domain/`, `app/orchestration/`, or `app/adapters/`.

### 6.2.2 State Contracts

Two distinct state types ŌĆö they MUST NOT be confused:

| Type | Owner | Lifetime | Mutability |
|---|---|---|---|
| **`CaseState`** (`orchestrator/state.py`) | Orchestrator only | One incident, full lifecycle | Mutable, single writer |
| **`AgentLocalState`** (`shared/agent_state.py`) | One agent subgraph | One subgraph invocation | Mutable inside the subgraph, dies on return |
| **`<X>Projection`** (frozen dataclass in `orchestrator/state.py`) | Built by orchestrator, read by agent | One subgraph invocation | **Immutable** |
| **`AgentEvent`** (frozen dataclass) | Returned by agent, applied by orchestrator | Persisted in `CaseState.events` | **Immutable** |

**Flow:**
1. Orchestrator builds `XProjection` from `CaseState`.
2. Orchestrator invokes agent subgraph with `{"projection": proj}`.
3. Agent runs internally on its own `AgentLocalState`.
4. Agent returns `final_event: AgentEvent`.
5. Orchestrator folds `AgentEvent` into `CaseState` (the **only** mutation point).

### 6.2.3 CaseState lifecycle

See `docs/diagrams/state-case-lifecycle.md`. Status enum:

```
NEW ŌåÆ INTAKE_OK ŌåÆ TRIAGED ŌåÆ TICKETED ŌåÆ NOTIFIED ŌåÆ AWAITING_RESOLUTION ŌåÆ RESOLVED
  Ōöé
  ŌööŌöĆŌ¢║ INTAKE_BLOCKED   (terminal)
  ŌööŌöĆŌ¢║ FAILED            (terminal, on agent.error)
```

### 6.2.4 Escalation policy

Lives in `orchestrator/router.py` ŌĆö pure functions only, no I/O. To change routing logic, edit this file and only this file.

| Routing decision | Function | Current policy |
|---|---|---|
| After intake | `route_after_intake` | `INTAKE_BLOCKED` ŌåÆ END, otherwise ŌåÆ triage |
| After triage | `route_after_triage` | `error` ŌåÆ END, otherwise ŌåÆ integration |
| After integration | `route_after_integration` | always ŌåÆ END (resolution is async) |
| `should_escalate(state, thresholds)` | governance-driven (ARC-017) | `kill_switch_enabled` ŌåÆ `True, "kill_switch"` \| `confidence < confidence_escalation_min` ŌåÆ `True, "low_confidence"` \| otherwise ŌåÆ `False, None` |

> **ARC-017 compliance:** `should_escalate()` signature changed from `(state)` to `(state, thresholds: GovernanceThresholds)`. The orchestrator fetches thresholds via `IGovernanceProvider.get_thresholds()` once at graph entry and passes the result to all routing functions that require it. Hardcoding threshold values in `router.py` is a rule violation.

### 6.2.5 Why two compiled graphs (DEC-006 / ARC-014)

The synchronous incident graph (`build_orchestrator_graph`) returns within bounded time ŌĆö its SLO is API response latency. Resolution depends on external human/system action of unknown duration, so it lives in a separate graph (`build_resolution_graph`) invoked by the resolution webhook handler. The two graphs share `CaseState` schema but never share a single invocation.

State persists in-process. Resets after a configurable cool-down. Every transition emits a structured log event (`triage.fallback_used`, `triage.degraded_mode`) and a Langfuse event.

Configuration (v4 ŌĆö DB-managed via `ILLMConfigProvider`, env var as fallback):
- `cb_failure_threshold` (default `3`) ŌĆö in `llm_config` table
- `cb_cooldown_seconds` (default `60`) ŌĆö in `llm_config` table
- `llm_timeout_s` (default `25`) ŌĆö in `llm_config` table

**Hot reload support (DEC-A06 / v4):**

The `LLMCircuitBreaker` exposes a `reconfigure(threshold: int, cooldown: int)` method that updates parameters in-place without losing the current failure count or state:
- Parameters update immediately at the next evaluation cycle.
- If the breaker is currently `OPEN`, it remains open until the previous cooldown expires, then uses the new cooldown for subsequent state transitions.
- The `container.reload_llm_adapter()` coroutine (asyncio.Lock-protected) atomically replaces the entire `LLMCircuitBreaker` singleton (including primary and fallback adapters) when the provider or model changes ŌĆö not just the parameters. This is the correct path when changing providers. `reconfigure()` is the lightweight path when only thresholds change.

## 11. Security Architecture ŌĆö Prompt Injection Defense (5 layers)

| Layer | What it does | Where |
|---|---|---|
| 1. Input sanitization | Strip control chars, zero-width Unicode. Length limit 8000. File allowlist. Size limit 5MB. | `app/security/input_sanitizer.py` |
| 2. Heuristic detection | Regex against known attack patterns | `app/security/prompt_injection.py` |
| 3. LLM judge | Cheap classification via `ILLMProvider.classify_injection()` | `app/security/prompt_injection.py` |
| 4. Output schema enforcement | Pydantic structured output via `ILLMProvider.triage()`. Free-form text never drives tool calls. | `domain/ports/llm_provider.py` + adapters |
| 5. Tool sandboxing | Adapters accept typed Pydantic args. No shell, no dynamic URLs. | `adapters/**/*.py` |

Blocked incidents are persisted with `blocked=true`, traced with `injection_detected=true`, and shown in the UI as "Blocked by guardrails ŌĆö see trace".

### 16.3 API contract (React ŌåÆ FastAPI)

| Parameter | Value |
|---|---|
| Base URL | `http://localhost:8000` (dev) / `VITE_API_BASE_URL` env var (prod) |
| Auth header | `Authorization: Bearer {jwt_token}` on all protected requests |
| Token source | `authStore.token` (from `localStorage`) |
| Content-Type | `application/json` |
| CORS | FastAPI `CORSMiddleware` allows origin `http://localhost:5173` (or `CORS_ALLOW_ORIGIN` env var) |

**Error handling contract:**
- HTTP 401 ŌåÆ clear token from `authStore` + `localStorage` ŌåÆ redirect to `/login`
- HTTP 403 ŌåÆ show inline 403 error page (do not redirect)
- HTTP 422 ŌåÆ surface validation errors in the form (field-level if possible)
- HTTP 503 ŌåÆ show "Service unavailable ŌĆö try again shortly" toast
- Network error ŌåÆ show "Cannot connect to server" toast

**Polling:** The dashboard polls `GET /incidents/{id}/status` every 30 seconds for in-progress incidents (SA-006). Implemented via TanStack Query `refetchInterval`.

## 18. Threshold Governance ŌĆö Where Each Threshold Lives

This table is the authoritative reference for where any threshold in the system MUST be stored. Violating the storage location is an ARC-017 violation.

| Threshold type | Examples | Storage location | Who may change it | Rationale |
|---|---|---|---|---|
| **CI eval gate** | `avg_overall_score >= 0.70`, `adversarial_recall == 1.0` | `ARCHITECTURE.md` ┬¦4.5 + env var `EVAL_MIN_SCORE` | `@architect` only | Architectural decision ŌĆö defines acceptable agent quality. Requires architect approval and doc update. |
| **Circuit breaker** | failure count, cooldown, timeout | env vars (`LLM_CIRCUIT_BREAKER_THRESHOLD`, `LLM_CIRCUIT_BREAKER_COOLDOWN_S`, `LLM_TIMEOUT_S`) | Operator (per environment) | Deployment parameter ŌĆö may vary between dev/staging/prod without code or architecture changes. |
| **Business governance** | confidence escalation, kill switch, severity routing, quality score, max RAG docs | PostgreSQL `governance_thresholds` + `GET/PUT /governance/thresholds` UI | Operator at runtime via UI | Runtime business decisions ŌĆö must be changeable without redeploy. Versioned via `updated_at` / `updated_by`. |

**Enforcement rule (ARC-017):** No business governance threshold may be defined as a Python literal, a pydantic-settings field, or an environment variable. The only exception is the `GovernanceThresholds` Pydantic model's field defaults ŌĆö these serve as safe fallback values when the DB is unreachable, and are not the operational defaults.

**v4 extension ŌĆö Config-from-DB (ARC-020):** The three-tier model above applies only to governance thresholds. For the full Config-from-DB principle covering ALL business configuration variables, see ┬¦4.14. The storage tier for all non-governance business config is `platform_config` or `llm_config` tables (not `governance_thresholds`).
